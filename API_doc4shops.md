# CallMy VPN — API документация для магазинов

**Версия:** 3.1.0  
**Базовый URL:** `https://callmy.ru/wp-json/vpn/v1/`

---

## Авторизация

Все запросы к магазинным эндпоинтам требуют параметр `api_key` — уникальный ключ вашего магазина.

Ключ выдаётся администратором в разделе **CallMy VPN → Магазины** и имеет формат:

```
shop_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

> ⚠️ Храните ключ в безопасном месте. При компрометации ключ можно сбросить в панели администратора — старый ключ немедленно перестаёт работать.

## Эндпоинты

### POST `/wp-json/vpn/v1/activate`

Активировать (продлить) подписку после успешной оплаты.

#### Параметры запроса

| Параметр       | Тип     | Обяз. | Описание |
|---------------|---------|-------|----------|
| `api_key`     | string  | ✅    | API-ключ магазина |
| `aid`         | string  | ✅    | Идентификатор устройства пользователя |
| `days`        | integer | ✅    | Количество дней подписки (> 0) |
| `order_id`    | string  | ✅    | Уникальный номер заказа в вашей системе |
| `amount`      | float   | —     | Сумма оплаты в рублях |
| `source`      | string  | —     | Источник/метка для аналитики |
| `note`        | string  | —     | Произвольный комментарий |

> **Идемпотентность:** повторный запрос с тем же `order_id` вернёт успех без повторной активации.

#### Пример запроса

```bash
curl -X POST https://callmy.ru/wp-json/vpn/v1/activate \
  -H "Content-Type: application/json" \
  -d '{
    "api_key":  "shop_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "aid":      "a1b2c3d4e5f6a7",
    "days":     30,
    "order_id": "PAY-20240614-001",
    "amount":   299.00,
    "source":   "pay.callmy.ru"
  }'
```

#### Успешный ответ `200 OK`

```json
{
  "ok":           true,
  "aid":          "a1b2c3d4e5f6a7",
  "days":         30,
  "bonus_days":   3,
  "active_until": "2024-07-14 12:00:00",
  "order_id":     "PAY-20240614-001",
  "shop":         "pay.callmy.ru"
}
```

| Поле           | Описание |
|---------------|----------|
| `days`        | Начислено дней |
| `bonus_days`  | Бонус рефереру (10% от `days`) |
| `active_until`| Новая дата истечения подписки |

#### Повторный запрос (already processed) `200 OK`

```json
{
  "ok":       true,
  "note":     "already processed",
  "order_id": "PAY-20240614-001"
}
```

### POST `/wp-json/vpn/v1/deactivate`

Отменить подписку для возврата средств. Откатывает начисленные дни.

#### Параметры запроса

| Параметр   | Тип    | Обяз.       | Описание |
|-----------|--------|-------------|----------|
| `api_key`  | string | ✅          | API-ключ магазина |
| `order_id` | string | ✅ или `aid` | Номер заказа для отмены |
| `aid`      | string | ✅ или `order_id` | Если нет `order_id` — отменяет последний платёж данного устройства |
| `reason`   | string | —           | Причина возврата (для внутренней аналитики) |

> Нужен хотя бы один из параметров: `order_id` или `aid`.  
> Магазин может отменить только **свои** платежи.

#### Пример запроса

```bash
curl -X POST https://callmy.ru/wp-json/vpn/v1/deactivate \
  -H "Content-Type: application/json" \
  -d '{
    "api_key":  "shop_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "order_id": "PAY-20240614-001",
    "reason":   "Пользователь запросил возврат"
  }'
```

#### Успешный ответ `200 OK`

```json
{
  "ok":               true,
  "aid":              "a1b2c3d4e5f6a7",
  "order_id":         "PAY-20240614-001",
  "days_removed":     30,
  "new_active_until": "2024-06-14 12:00:00",
  "shop":             "pay.callmy.ru"
}
```

| Поле               | Описание |
|-------------------|----------|
| `days_removed`    | Сколько дней снято |
| `new_active_until`| Новая дата истечения подписки |

> Если после отмены дата активной подписки оказывается в прошлом — статус меняется на `expired`.  
> Бонусные дни, начисленные рефереру при оплате, также откатываются автоматически.

## Коды ошибок

| HTTP | `error_code`       | Описание |
|------|--------------------|----------|
| 401  | `missing_api_key`  | Параметр `api_key` не передан |
| 403  | `invalid_api_key`  | Неверный или отключённый ключ |
| 403  | `forbidden`        | Платёж принадлежит другому магазину |
| 400  | `bad_request`      | Отсутствует обязательный параметр (`aid`, `days`) |
| 404  | `not_found`        | Устройство или платёж не найдены |

#### Пример ответа с ошибкой

```json
{
  "code":    "invalid_api_key",
  "message": "Invalid or inactive api_key",
  "data":    { "status": 403 }
}
```

## Получение AID пользователя

`aid` — уникальный идентификатор устройства пользователя в системе CallMy. Пользователь видит его в приложении в разделе «Профиль».

При создании платежа магазин должен получить `aid` от пользователя, например:

- Поле ввода «Профиль ID» на странице оплаты
- Передача через deeplink / параметр URL

---

## Алгоритм интеграции

```
1. Пользователь выбирает тариф на сайте магазина
2. Вводит свой Профиль ID (aid) из приложения CallMy
3. Магазин принимает оплату
4. При успешной оплате — POST /vpn/v1/activate
5. При возврате средств  — POST /vpn/v1/deactivate
```

## Тестирование

Проверьте авторизацию (ожидается 403 без ключа):

```bash
curl -X POST https://callmy.ru/wp-json/vpn/v1/activate \
  -H "Content-Type: application/json" \
  -d '{"aid":"test","days":1}'
```

Ожидаемый ответ:
```json
{"code":"unauthorized","message":"api_key required","data":{"status":401}}
```
